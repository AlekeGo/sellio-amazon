# Day 13.3 — Compact Report

## Goal

Simplify the AI audit report into a compact seller-friendly format readable in 30–60 seconds.

## compact_report structure

The Gemini response now includes a `compact_report` object alongside all existing fields.
It is stored inside `AuditResult.concise_report` (JSON field) as `concise_report.compact_report`.
No new database migrations are needed.

```json
{
  "compact_report": {
    "score_snapshot": {
      "overall_score": 67,
      "status": "Needs work",
      "main_problem": "Weak product positioning",
      "quick_win": "Rewrite title and bullets first",
      "sub_scores": {
        "seo": 61,
        "copy": 72,
        "images": 54,
        "trust": 58,
        "competitor_position": 63
      }
    },
    "fix_first_table": [
      {
        "problem": "Weak title",
        "why_it_matters": "Buyers do not understand the value fast",
        "fix": "Use the new copy-ready title"
      }
    ],
    "buyer_and_competitor_insights": {
      "buyer_objections": [
        {
          "buyer_concern": "Is the quality good?",
          "fix": "Add proof of quality in bullets and images"
        }
      ],
      "competitor_actions": [
        {
          "competitor_wins_in": "Better image story",
          "your_action": "Add benefit infographic and lifestyle image"
        }
      ]
    },
    "next_actions": [
      {
        "step": 1,
        "action": "Copy the new title",
        "priority": "High"
      }
    ],
    "advanced_details": {
      "keywords": ["keyword phrase"],
      "a_plus_content_plan": ["short plan item"],
      "detailed_notes": ["short note"]
    }
  }
}
```

## Score rules

- `overall_score` must equal the top-level `score` field.
- All scores (overall + 5 sub-scores) must be **varied integers 0–100**.
- Never round to multiples of 5 or 10.
- Good examples: 47, 52, 67, 73, 81.
- Bad examples: 20, 40, 60, 80.

## Visible report order (compact path)

When `compact_report` is present in the Gemini response:

1. **ReportHeader** — product name, overall score ring, score label, Seller Persona badge (unchanged)
2. **Score Snapshot** card — status label, main_problem, quick_win, 5 sub-score bars
3. **Fix This First** table — 3 rows × 3 columns (Problem | Why It Matters | Fix)
4. **Pro Upgrade Pack** — "Your Pro Upgrade Pack is ready" with tab nav (Title | Bullets | Description | Details | Image Plan) + Copy Full Pack + Image Studio buttons
5. **Buyer & Competitor Insights** — two inline tables: Buyer Concern/Fix + Competitor Wins In/Your Action (max 3 rows each)
6. **Image Plan** table — 4 columns (Image | Goal | Headline | Action), max 6 rows, Open Image Studio button; sourced from `concise_report.image_gallery_plan`
7. **Next Actions** — numbered list max 5, with priority badges
8. **Advanced Details** — collapsible section, closed by default (keywords, A+ plan, notes)
9. **Image Studio CTA** banner

## Length rules

| Section | Limit |
|---|---|
| fix_first_table | max 3 rows |
| buyer_objections | max 3 rows |
| competitor_actions | max 3 rows |
| next_actions | max 5 steps |
| advanced_details.keywords | max 8 |
| advanced_details.a_plus_content_plan | max 5 |
| advanced_details.detailed_notes | max 5 |

## Pro Upgrade automatic behavior

`pro_upgrade_pack` is generated by Gemini in the same call as the audit (no separate action required).
The frontend reads it from `AuditResult.pro_upgrade_pack` (already stored separately in the DB).
The compact view shows a "Your Pro Upgrade Pack is ready" banner above the existing ProUpgradePackBlock.

## Backward compatibility

| Condition | Display |
|---|---|
| `concise_report.compact_report` exists | CompactAuditReport (new) |
| `report_version === 'v2'` and no `compact_report` | ConciseAuditReport (existing) |
| `report_version !== 'v2'` | AuditReport (legacy) |

Old reports are not affected. No migration is needed.

## Pro Upgrade Pack — tab layout

The ProUpgradePackBlock now shows a tab bar instead of all sections expanded at once:

| Tab | Content |
|---|---|
| Title | Copy-ready title with Copy Title button |
| Bullets | Numbered bullet list with per-bullet copy |
| Description | Full description with Copy Description button |
| Details | Product details fixes (field → value) |
| Image Plan | Image briefs table (Image | Headline | Action) |

Tabs that have no content are hidden automatically.
"Copy Full Pack" and "Open Image Studio" buttons are always visible in the header.

## Fallback behavior

| Condition | Displayed component |
|---|---|
| `concise_report.compact_report` exists | `CompactAuditReport` (new compact layout) |
| `report_version === 'v2'` + no `compact_report` | `ConciseAuditReport` (legacy v2, deduped) |
| `report_version !== 'v2'` | `AuditReport` (legacy v1) |

In the `ConciseAuditReport` fallback:
- "Top Problems" section removed (Fix This First covers it)
- "Ready-to-Copy Listing Upgrade" hidden when `pro_upgrade_pack` is present (content already in ProUpgradePackBlock)
- Standalone "Product Details Fixes" hidden when `pro_upgrade_pack` is present

## Mobile behavior

- All tables use `gridTemplateColumns` with fixed fractions — narrow on mobile but readable
- `flexWrap: 'wrap'` on all row-level containers ensures overflow wrapping
- Copy buttons use `flexShrink: 0` to prevent truncation
- Tab bar uses `overflow: 'auto'` so tabs scroll horizontally on narrow screens
- ProUpgradePackBlock header stacks vertically on mobile via `flexWrap: 'wrap'`
